# 知识库入口：它有什么意义

> 更新时间：2026-06-19

这个知识库的意义，不是把资料堆起来，而是把你这台 Mac 上分散的项目、环境、风险和下一步动作，整理成一个可以持续使用的工作系统。

## 它解决什么问题

### 1. 不再靠记忆找项目

以前要靠脑子记：哪个目录有什么项目、哪个是前端、哪个是后端、哪个能跑、哪个很旧。

现在有了：

- `cuizihao 目录代码项目盘点`
- `CETC Project 项目盘点`

它们相当于本机代码地图。以后你想知道“这个电脑上到底有哪些项目”，不用重新翻目录。

### 2. 不再盲目接手老项目

老项目最怕一上来就 `npm install`、`npm run dev`，然后报一堆错，不知道是自己环境问题，还是项目本来就坏。

现在有了：

- `CETC 本机开发环境基线`
- `CETC 重点项目运行环境复现计划`
- `CETC 重点项目复现记录表`

它们记录了 Node、npm、Java、Maven、锁文件、构建结果和失败原因。下次再跑项目，可以按记录来，不用重踩坑。

### 3. 风险被放到台面上

老项目里常见的问题包括：

- 内网地址写死
- 配置文件里有敏感信息
- 依赖太老
- 没有锁文件
- 构建脚本和目录假设不一致
- 业务项目没人敢动

现在有了：

- `CETC 项目健康度报告`
- `CETC 敏感配置复核清单`
- `CETC 项目整理路线图`

这些文档让风险从“隐约觉得有问题”变成“明确知道在哪里、优先处理什么”。

### 4. 能形成交接材料

如果以后你要交接、汇报、带新人、或者隔很久后自己回来继续做，这个知识库能直接回答：

- 项目有哪些？
- 哪些项目最重要？
- 哪些项目能跑？
- 哪些项目跑不起来？
- 跑不起来的原因是什么？
- 下一步应该先修什么？

这比临时口头解释可靠得多。

### 5. 它会慢慢变成你的个人工程资产

短期看，它是一个项目文档站。

长期看，它会变成：

- 你的本机资产清单
- 项目接手手册
- 环境复现手册
- 技术债账本
- 安全风险清单
- 个人工作履历的一部分

每多记录一次真实验证结果，它就更有价值。

## 现在应该怎么看

如果只想快速了解全局，按这个顺序看：

1. `CETC Project 项目盘点`
2. `CETC 项目健康度报告`
3. `CETC 项目整理路线图`

如果要真的开始跑项目，按这个顺序看：

1. `CETC 本机开发环境基线`
2. `CETC 重点项目运行环境复现计划`
3. `CETC 重点项目复现记录表`

如果担心公开仓库里的敏感信息，先看：

1. `CETC 敏感配置复核清单`

## 当前最有价值的结论

目前已经确认：

- `xinfang-web-admin` 可以在 Node 12 下安装并生产构建成功。
- `cetc-moniwa-ui` 可以安装，但构建失败点在 `regenerator-runtime/runtime.js`。
- `bi-ui` 可以安装，但 VuePress 文档构建失败点在 `stylus-loader`。
- `jun-dd-web` 可以安装，但构建失败点包括缺 `clipboard`、依赖版本过新和 `tar` 目录假设不一致。
- Node 8.9.4、10.24.1、12.22.12、14.21.3 和 Maven 3.9.9 已经补齐。

这些不是抽象总结，而是已经在这台机器上实际验证过的结果。

## 下一步最值得做什么

最值得做的不是继续盲扫更多项目，而是把 4 个重点项目推进到“可稳定复现”：

1. 给 `xinfang-web-admin` 补 `.nvmrc` 和运行说明建议。
2. 修 `cetc-moniwa-ui` 的依赖完整性问题。
3. 固定 `bi-ui` 的 VuePress / Stylus 依赖组合。
4. 补齐 `jun-dd-web` 的 `clipboard` 依赖，并固定过新的前端依赖版本。

做到这一步后，知识库就不只是记录问题，而是开始驱动问题被解决。
